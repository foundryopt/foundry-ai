# Guardrails — What Not To Do

Seven years of lessons learned. The devil is in the details.

These aren't theoretical risks — they're things that actually went wrong, cost real money, 
or burned real time on SHB projects. The playbook tells you what to do. 
The guardrails tell you what will kill you if you're not watching.

## Structure

| File | Covers |
|------|--------|
| `development.md` | Deal selection traps, zoning pitfalls, market timing |
| `design.md` | Code gotchas, self-cert audit risks, coordination gaps |
| `construction.md` | Sub failures, schedule killers, site condition surprises |
| `procurement.md` | Lead time disasters, spec mismatches, vendor traps |
| `finance.md` | Cash flow timing, draw schedule errors, exit miscalculations |
| `ops.md` | Insurance gaps, lien waiver mistakes, compliance oversights |
| `people.md` | Hiring mistakes, sub red flags, investor warning signs |

## How to Read These

Each entry follows this format:

```
### [Short name]
**What happened:** One-line description
**What it cost:** Time, money, or opportunity lost
**The red flag we missed:** What we should have seen earlier
**The rule now:** What we always do (or never do) because of this
```

## Contributing

After every project closeout, do a guardrails review:
- What went wrong that we didn't expect?
- What went right because of an existing guardrail?
- What guardrail would have prevented the biggest headache?

Add entries. Version control catches the evolution.

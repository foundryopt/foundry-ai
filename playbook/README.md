# SHB Playbook

The codified operating system for SHB Group's Design-Build-Development model.

## Purpose

This playbook encodes how SHB thinks, decides, and executes — so that a 3-person core team 
can run $10M+ in concurrent projects without losing signal.

Every file here answers one question: **"What would Kuan decide?"**

## Structure

```
playbook/
├── development/     # Deal sourcing → acquisition → zoning → exit
├── design/          # Schematic → DD → CD → permit (self-cert flow)
├── construction/    # Pre-con → GMP → field execution → closeout
├── procurement/     # Spec → bid → PO → delivery → install
├── finance/         # Pro forma → draw schedule → AP/AR → refi/exit
└── ops/             # Team, subs, insurance, compliance
```

## How to Use

1. **Starting a new project?** → `development/acquisition-checklist.md`
2. **Setting up a pro forma?** → `finance/pro-forma-template.md`  
3. **Kicking off construction?** → `construction/pre-con-checklist.md`
4. **Onboarding a sub?** → `ops/sub-onboarding.md`

## Principles

- **Repeatable > Exceptional.** The playbook optimizes for the 80% case. Exceptions get noted, not enshrined.
- **Decisions travel without Kuan.** If someone needs to call Kuan to make a routine decision, the playbook has a gap.
- **Version-controlled.** Every project sharpens the playbook. After closeout, lessons learned get merged back.
- **Machine-readable where possible.** Viktor (AI) references these files in real-time. Clear structure > prose.
